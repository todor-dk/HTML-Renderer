Configuring Outlook

Normally Outlook requires minimal setup using the Outlook Autodiscover configuration option. However, if you encounter problems with this or wish to configure additional options, you may set Outlook up manually. When setting up Outlook at home for the first time, please connect to VPN. If you are using Outlook for the first time, no profile will exist and you will be creating a new Exchange profile.

  • Your e-mail address will remain unchanged. People may still send mail to your netid@ucr.edu and first.last@ucr.edu e-mail addresses.

Using Outlook Autodiscover Configuration

  • Start Outlook.

If no Profile Exists

  • You will be prompted to create an Exchange connection using the Add New E-mail Account wizard.

Outlook 2007 Profile Exists

  • If a profile does exist, navigate to Tools -> Account Settings... and then click New. Select the Microsoft Exchange option and then click "Next". 

Outlook 2010 Profile Exists

  • If a profile does exist, click on the 'File' tab -> 'Info' -> click the 'Add Account' button. Please remove the old webmail/mirapoint account, as it no longer exists.
  • To utilize the Autodiscover configuration technologies, enter the account name, e-mail address and account password.
    • The e-mail address will be in the form UCRNetID@ucr.edu.
  • Click "Next" to continue.
  • Outlook will attempt to determine your connection settings. This process may take several minutes.
  • You will be prompted for your account credentials in a Windows Security screen.
  • Select Use Another Account
  • The username will be the domain 'ads' with a backslash (\) and your UCR Net ID. In the example, the username is ADS\gboole.
  • Enter the associated account password and click "OK".
  • Note: The Remember my password feature will not work for security reasons.
  • Once the connection has been verified, you will be notified of the successful configuration. Click Finish.
  • Note: Autodiscover cannot set every configuration - please check to see if both FAST and SLOW networks are set to use HTTP first under the Proxy Settings (see Manual Configuration).

Outlook Manual Configuration

  • Start Outlook.
    • If no profile exists, you will be prompted to create an Exchange connection using the Add New E-mail Account wizard.
    • If a profile does exist, navigate to Tools > Account Settings... and then click New. Select the Microsoft Exchange option and then click "Next". Please remove the old mirapoint/webmail account.
  • Check the option "Manually configure server settings or additional server types" and click "Next".
  • Select the "Microsoft Exchange" service and then click "Next".
  • Enter the Microsoft Exchange server name post.ucr.edu. Ensure the "Use Cached Exchange Mode" option is checked.
  • Enter the username, i.e. your UCR Net ID. A new window will come up and ask you to authenticate.
    • Username: ads\UCRNetID
    • Password: your UCR Net ID password
  • Click the "More Settings..." button.
  • Select the "Connection" tab and check the option "Connect to Microsoft Exchange using HTTP".
  • Click the "Exchange Proxy Settings..." button.
  • Enter the name post.ucr.edu in the http:// section.
  • Ensure that both fast and slow networks use HTTP first.
  • Verify that the authentication method is NTLM.
  • Click "OK" until you return to the Microsoft Exchange Settings window, and then click "Next".
  • You may be notified that Outlook must be restarted. Click "OK".
  • Once Outlook has been configured, you will be congratulated. Click Finish.

Connecting to Exchange

  • Start Outlook.

  • You will be prompted for your account credentials in a Windows Security screen.

  • Select The ads\your UCR Net ID account

  • Enter the associated account password and click "OK".

  • Note: The Remember my password feature will not work for security reasons.
  • The first time Outlook connects to the Exchange system, you will be notified that it is being prepared.
  • You may be prompted if you would like to use the Outlook search feature. On some systems, this must be downloaded and installed.
  • The main Outlook window will appear and begin synchronization with the Exchange server. For an existing mailbox with a significant size, this process can take some time to complete, please be patient. Once the local and server mailboxes are synchronized, only changes are sent back and forth.

Junk Mail Filtering

  • Go to Actions (or the HOME tab) ->Junk (E-Mail)->Junk E-Mail Options.  A new window will appear.  Set the option for "No Automatic Filtering."  Press OK to return to Outlook.
  • Next log onto OWA at http://post.ucr.edu
  • At the top right of the screen, click on options. Along the left column, click on "Block or Allow."  Change the setting to "Don't move mail to my Junk E-Mail folder." Press the save button on the bottom right of the screen.

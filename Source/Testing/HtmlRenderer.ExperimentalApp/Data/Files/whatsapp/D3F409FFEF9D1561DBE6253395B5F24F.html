Because "WhatsApp does not want to inundate users with messages they don't want to see", per Brian Acton in the latest F8 conference. While that may be sad for developers, I think it's a good decision.

WhatsApp's appeal is private, human, group conversations.

It does that extremely well and for more than a billion people. Disturbing that would be risky.

Maybe Im too late to answer on this, but like the I-phone, android apps can also communicate with WhatsApp.

WhatsApp FAQ - Im an Android developer, how can I integrate WhatsApp with my app?

But the thing is we can make our app send information to WhatsApp, not the other way around. Either using the android intent system to open another application directly or using the system picker or maybe using a custom URL provided by WhatsApp.

Since WhatsApp isnt providing an API, we can think of accessing the its database on the android system. But the problem to get meaningful information is difficult because the data is encrypted.

There is a blog on decryption of WhastApp data using an android app or using python code (I have not tried this, cant confirm if it works yet) : How To Read Someone's WhatsApp Database Messages

Unless WhatsApp team provides a public API to use their services, there can never be a legal way of using it. But the question is making it secure (end to end encryption) as well as giving access to use the same service in a bit of a problem, this might be the reason for not giving out an API.

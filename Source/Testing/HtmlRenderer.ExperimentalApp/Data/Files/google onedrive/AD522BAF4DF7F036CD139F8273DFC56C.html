Same experience here.

OneDrive has been using over a week now just looking for what to upload of my 600 GB folder, before it finally started actually uploading yesterday. I know it has been doing that, because it has been using 25% CPU the entire week, so I didn't give up.

I experienced about three hangs from OneDrive when first starting out (file name too long, file too big and one where it simply wouldn't upload). All those times I had to download the troubleshooter...

Obviously it's buggy, because it should absolutely be able to resume and refresh itself either manually or automatically when there's problematic files. It shouldn't stop the entire 600 GB file transfer and simply freeze because of one long file name or one big file...

But move or rename the problematic files, run the troubleshooter a few times, and you should be back on track.

If I had to choose, I'd rather use Google Drive or Dropbox, but I got the 1 TB storage with my 4-year Office 365 student subscription at about 60 $ or so, so there's no reason to waste that offer.

Rather a little bit of fiddling than money out the window, frustrating as it might be at times.

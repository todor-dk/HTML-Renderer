It would be useful to understand how the formula came about. They are basically two different scales and there is a linear relationship between the two scales. It means if I plot the temperature in C and F, they are going to fall in a straight line. The formula is nothing but the equation for the straight line.

Water boils at 100 C in the Celsius scale and when observed in the Fahrenheit scale it is 212. This is one set of points. Similarly if you observe the freezing point, it is 0 C and 32 F. This is the next set of points. So you have a c1, f1 and c2, f2

Equation of the line connecting C and F would be F = mC + b
Eqn 1: 32 = 0*m + b; thus b = 32
Eqn 2: 212 = 100m + 32; Thus m = (212-32)/100 = 9/5

Hence the formula; F = (9/5)*C + 32

The conversion from C to F requires you to multiply the temperature in C by 1.8 (each change of 1 degree C represents a change of 1.8 degrees F) and add the offset of 32 (32 degrees F = 0 degrees C).

A quick way to do it in your head is “double it and add 30”. So 20 degrees C would be about 2*20 + 30 = 40+30 = 70 degrees F. The actual answer is 68, but 70 is close. The conversion is exactly right at 10 degrees C (50 degrees F), but diverges from there, so the further you are from 10 degrees C, the less accurate this estimation is.

In C using floating point (doubles), the most accurate conversion would look something like this:

double CtoF(double C)
{
   return (C*1.8) + 32.0;
}

If you need to work with integers, the following should work:

int CtoF(int C)
{
   int round = 25;

   if (C < 0)
   {
      round = -round;
   }

   return ((C*90)+round)/50 + 32;
}

This will work for the range of about -360 to +360 degrees C on a 16-bit machine, or -23860000 to +23860000 degrees C on a 32-bit machine. The variable round is used to round the value to the nearest decimal (integer division in C truncates).
